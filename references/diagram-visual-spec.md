# 工作图表视觉规范 · Geist 绿白

> 适用：飞书画板（beautiful-feishu-whiteboard / lark-cli）里产出的**工作型图表**——全景图 / 骨架图 / 路线图 / 机制图等。理念源自 Geist Design System / Vercel Design。
> **一句话原则：让位于内容——用层级、留白、细边框和一个绿色组织信息，不用重色/阴影/装饰。**

## 一、调色板（只用这些 token，禁裸配色）

**中性灰阶（承担 90%）**
- 页底 `#FAFBFB`｜卡面 `#FFFFFF`｜卡片边框 `#E1E3E4`（2px）
- 标题字 `#0A0C0C`｜正文/卡内字 `#1A1E1F`｜辅助字 `#6B7173`｜连线/箭头 `#4A5052`

**品牌绿（唯一强调色，单色）**
- 主绿 `#10A37F`（标题条、数据条、中枢节点、强调）｜浅绿底 `#E7F6F0`｜绿字 `#0B7355`
- 规则：**一张图只让绿色做"主路径/已确认/最重要"**；绿色出现 = 这里重要。

**四档状态语义色（浅底 + 同色描边 + 深字）**

| 状态 | 底 | 描边/点 | 用途 |
|---|---|---|---|
| 已有方案/已确认 | `#E7F6F0` | `#10A37F` | 收口的 |
| 已共识 | `#E9F1FC` | `#3B7DE0` | 达成一致 |
| 待共创 | `#F0ECF9` | `#7E57C2` | 与业务共创 |
| 待拍板/待办 | `#FCF2E1` | `#E8A23D` | 需决策（用琥珀，不用红） |
| 真异常（少用） | `#FBE9E7` | `#D9534F` | 仅"错/风险"才用红 |

文字在浅语义底上要强调时用对应深字：绿 `#0B7355` / 蓝 `#2C5FB0` / 紫 `#7E57C2` / 琥珀 `#8A5B12`。

**分类色（当颜色表"类别"而非"状态"，如分支/分场）**：绿 `#10A37F` / 蓝 `#3B7DE0` / 琥珀 `#E8A23D` / 紫 `#7E57C2` / 中性深 `#0A0C0C`（中枢或首尾用）。

## 二、视觉规则
1. **边框优先于阴影**：静态卡片**不用阴影**，用 2px `#E1E3E4` 边框分隔；深度靠留白和层级。
2. **标题条用主绿** `#10A37F` + 白字；卡面白、页底近白。
3. **浅底深字、深底白字**；浅语义底上强调用对应深色字。
4. **直角或统一小圆角**（rx 0–8，同图一致）；间距走 4px 栅格、留白充足。
5. **一图一主强调**：绿色只给最重要的一个点争注意，其余中性。

## 三、飞书画板的硬限制（必须遵守）
- 只用原生形状（rect/circle/line/text），**不设 `font-family`**。
- 箭头一律 `marker-end` 原生连接器，**禁手画三角/polygon**。
- **禁** gradient / filter / opacity / 模糊；本规范默认**不用阴影**。
- **绝不要在画板 SVG 文字里放 emoji**（⭐🚀🎯⚠️🅿️ 等 pictographic）——whiteboard-cli 渲染遇 emoji 会在该处**静默中断、后续元素全不画**（PNG 只出半张却报 error:""、--check warnings:0，极难发现）。用色块/底色/①②③ 圈号替代。
- 导出 PNG 的**文字颜色不可靠**（白字常变黑）——核验颜色用 `--output_as raw` 或看线上，别信导出图。

## 四、四类图的固定范式
- **全景图**：分层铺（地基 → 支柱 → 运营 → 终局），每块按四档状态上色，底部放绿色硬数据条。
- **骨架图**：中枢（绿）+ 主干（分类色）的浅层导图，**只到模块层、不画细节**，做"讨论锚点"。
- **路线图**：横向阶段/分场卡片流，首尾用中性深、中段用分类色，产出用语义 tag。
- **机制图**：左右或上下对照 + `marker` 箭头，突出"一个机制怎么运转"。

## 五、产出流程
作 SVG（逻辑宽 ~1500–1700）→ `whiteboard-cli -i x.svg -o x.png -f svg` 渲染 → 看图修（溢出/对齐/边距/半张图）→ 推送 `whiteboard-cli --to openapi | lark-cli whiteboard +update`。

换肤脚本见本仓库 `scripts/recolor_geist.py`（状态/结构型，含阴影删除+语义映射）与 `scripts/recolor_geist_cat.py`（分类型）——把 linen 配色的画板批量转成 Geist。
